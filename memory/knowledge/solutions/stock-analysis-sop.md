## 二、错误根因分析

### 流程漏洞1：支撑判断缺少验证步骤
```
❌ 错误流程：看到低点 → 看到后续价格在该位置上方 → 判断为支撑
✅ 正确流程：看到低点 → 检查后续是否被击穿 → 若未被击穿或迅速收回 → 判断为支撑
```

**关键遗漏**: 没有检查1月30日走势，64.23在次日就被击穿

### 流程漏洞2：新高判断缺少历史对比
```
❌ 错误流程：看到今日涨停 → 直觉判断为新高
✅ 正确流程：看到今日价格 → 对比历史所有高点 → 确认是否新高
```

**关键遗漏**: 没有对比1月28日的71.24盘中高点

### 思维漏洞：先有结论再找证据
- 看到64.23附近开盘 → 大脑自动关联"支撑"
- 看到涨停 → 大脑自动关联"新高"
- 这是典型的**确认偏误**(Confirmation Bias)

## 三、修正后的分析流程（闭环版）

### Step 1: 数据收集（事实层）
```
□ 获取完整K线数据（开盘、收盘、最高、最低、成交量）
□ 标注所有重要价格节点（前高、前低、涨停、跌停）
□ 检查数据完整性（是否有缺失日期）
□ 记录关键日期的分时特征（长上影、长下影、大阴线、大阳线）
```

### Step 2: 数据验证（验证层）
```
□ 支撑位验证：
  - 该价位是否被后续走势测试过？
  - 测试时是支撑住了还是被击穿？
  - 若被击穿，是否有迅速收回？
  
□ 压力位验证：
  - 该价位是否多次触及后回落？
  - 是否有放量突破的历史？
  
□ 新高/新低验证：
  - 当前价格与N日内的最高/最低对比
  - N的取值：短期(5日)、中期(20日)、长期(60日)
```

### Step 3: 逻辑推导（推理层）
```
□ 支撑/压力的有效性必须基于数据验证结果
□ 趋势判断必须基于价格结构，而非单一K线
□ 量价关系必须匹配（放量突破/缩量回调）
□ 避免直觉判断，所有结论必须有数据出处
```

### Step 3.5: 政策/宏观分析（如涉及）
```
□ 全球宏观背景：美元指数、美债收益率、地缘政治
□ 主要经济体联动：美国、欧洲、日本、中国的政策互动
□ 时间窗口分析：选举周期、央行会议、财报季
□ 资本流向：全球资金从哪来、到哪去
□ 政策比较：同类政策在历史/其他国家的后果
□ 避免"政策孤岛"：不孤立看单一政策，要放在全球棋盘上
```

### Step 4: 输出校验（闭环层）
```
□ 反推检验：如果我的结论错了，数据会是什么样的？
□ 反例检验：是否存在数据可以推翻我的结论？
□ 数据溯源：结论中的每个数据点是否都被验证过？
□ 逻辑闭环：从数据到结论的推导是否无跳跃？
```

## 四、股票分析检查清单（Checklist）

每次分析前，必须完成以下检查：

### 基础数据检查
- [ ] 已获取至少20日K线数据
- [ ] 已标注所有关键价位（前高、前低、缺口）
- [ ] 已检查是否有重要日期缺失

### 支撑/压力检查
- [ ] 每个支撑位都验证过是否被击穿
- [ ] 每个压力位都验证过是否多次受阻
- [ ] 支撑/压力的强度有数据依据

### 趋势判断检查
- [ ] 新高/新低有历史数据对比
- [ ] 趋势判断基于价格结构而非单根K线
- [ ] 量价关系已分析

### 输出质量检查
- [ ] 所有结论都有数据出处
- [ ] 不存在"可能""也许"等模糊表述
- [ ] 风险点已明确列出

### 政策分析检查（如涉及）
- [ ] 已梳理全球宏观背景（美元、美债、地缘政治）
- [ ] 已分析主要经济体的联动反应（美国、欧洲、亚洲）
- [ ] 已考虑政策的时间窗口（选举、会议、季节性因素）
- [ ] 已识别政策与全球资本的流向关系
- [ ] 政策分析不孤立，有国际比较视角

## 五、常见错误模式（警示）

| 错误模式 | 表现 | 修正方法 |
|----------|------|----------|
| **确认偏误** | 先有结论再找证据 | 强制先列数据再推导 |
| **近因效应** | 被最新数据干扰判断 | 拉取足够长历史数据 |
| **锚定效应** | 过度关注某个价格 | 建立完整支撑压力体系 |
| **过度自信** | 忽略反例 | 强制寻找证伪证据 |
| **政策孤岛** | 只分析政策本身，忽略全球背景 | 必须结合全球宏观形势 |
